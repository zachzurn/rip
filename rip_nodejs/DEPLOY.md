# Deploying rip-receipt to npm

Native Node.js addon. The npm package contains a prebuilt `.node` binary ŌĆö users don't need Rust.

## Prerequisites

- [Rust](https://rustup.rs/) (stable)
- Node.js 18+
- npm account with publish access to `rip-receipt`

## Build

```bash
cd rip_nodejs

# Install napi CLI (dev dependency)
npm install

# Build native addon (release mode)
npm run build
```

This produces:
- `rip_nodejs.node` ŌĆö native binary (platform-specific)
- `index.d.ts` ŌĆö TypeScript declarations (auto-generated by napi-rs)

`index.js` is hand-written and checked in (ESM loader for the `.node` file).

## Test

```bash
# All renderers ŌĆö HTML, text, PNG, raster, ESC/POS
node test_smoke.mjs

# Resource loading ŌĆö local images, caching, ESC/POS with resources
node test_resources.mjs
```

## Publish

```bash
# 1. Bump version in package.json
#    (also bump Cargo.toml version to match if you want them in sync)

# 2. Build release
npm run build

# 3. Run tests
node test_smoke.mjs && node test_resources.mjs

# 4. Verify what will be published
npm pack --dry-run
```

The dry-run should show exactly these files:
```
index.js            ESM loader
index.d.ts          TypeScript types
rip_nodejs.node     Native binary
README.md           npm page
package.json        Package metadata
```

```bash
# 5. Publish
npm publish
```

## Platform notes

The `.node` binary is platform-specific. A package built on Windows x64 only works on Windows x64. Right now we publish a single-platform build.

For multi-platform distribution in the future, napi-rs supports the `--platform` flag which generates separate `@rip-receipt/win32-x64-msvc`, `@rip-receipt/linux-x64-gnu`, etc. packages with a root package that auto-selects the right one. See the [napi-rs docs](https://napi.rs/docs/cross-build/summary) for CI setup.

## Troubleshooting

**"Cannot find module './rip_nodejs.node'"** ŌĆö run `npm run build` first.

**Tests fail after rebuild** ŌĆö make sure you built in release mode (`npm run build`, not `npm run build:debug`). Debug builds are slower but should produce identical output.

**napi deprecation warning** ŌĆö `napi.name is deprecated, use napi.binaryName instead` ŌĆö this is a cosmetic warning from the napi CLI. It still works. To silence it, rename `"name"` to `"binaryName"` in the `napi` section of `package.json`.
